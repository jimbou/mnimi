Abstract

Naive caches turn every repeated prompt into the same response. That saves time—but silently invalidates Pass@k, uncertainty estimates, repair loops, retries, and other probabilistic workflows. Mnimi makes the required sampling semantics explicit and composable.

The tension

The same prompt can need the same answer—or a fresh one.

Caching is not just a storage decision. In stochastic programs, it changes the meaning of a sample. Mnimi lets each algorithmic scope say which meaning it needs.

The design pattern

Statistical intent, composed like decorators.

Every model exposes one small interface: sample(prompt) → Iterator[str]. Wrapping the reference changes how sequences are consumed—not the workflow that consumes them.

Stability within a scope

Repeatable returns the same independent sequence for matching prompts, starting from its beginning on every call.

02

Diversity across scopes

Independent shares and advances iterator state, so later calls consume responses not used by earlier calls.

03

Continuity across runs

Persistent records response sequences on disk, enabling deterministic replay and incremental experiments.

The implementation

A small cache with a precise contract.

The reference Python implementation is intentionally easy to adopt: one file, no dependencies, and one sampling interface that works across providers.

agentic_workflow.py
# Persist responses across program runs
model = Persistent(model, "~/.llm_cache")

for attempt in range(NUM_RETRIES):
    # Fresh response sequence for every retry
    independent = Independent(model)

    # Stable responses inside this attempt
    repeatable = Repeatable(independent)
    result = run_pipeline(repeatable)
    if result.valid:
        break
